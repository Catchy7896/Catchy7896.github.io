2 发布文章

①scaffolds\post.md:文章的模板
②source\ _posts：在这里可以看到发的文章，进行内容编辑，也可以删除不想要的文章。
③public：存放的根据md生成的页面

注意：
每次更改数据后都要执行下面三个命令：

  • 清除缓存文件:hexo clean
  • 生成静态文件:hexo g = hexo generate
  • 上传到github：hexo deploy =hexo d

3 一些注意事项

  • 出现文章进入网页后无法预览的情况：检查根目录_config.yml配置文件的permalink参数后面的title是否有/:permalink: :category/:title/

  • 文章不能有跳级（一级标题下面有三级标题又有一个二级标题）

  • 要使用hexo new page tags about来创建标签和关于的文件夹，否则没有索引，会显示空白，读不到样式。
